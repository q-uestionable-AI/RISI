# Public engineering roadmap

This page is a non-authoritative engineering summary. The authoritative research charter and
approval state are maintained separately in the private research vault.

## Bootstrap

- establish packaging, quality gates, CI, schemas, and documentation boundaries;
- keep the engine and adapters implementation-free;
- verify cross-platform installation and CLI behavior.

## M0: environment and claim verification

- verify live lab capacity and documentation;
- freeze definitions, scope, ethics, cost caps, and initial scenario protocol;
- record decisions without publishing private infrastructure or research material.

## M1: reference substrate

- implement disposable complete-state storage;
- add pure-read and intentionally adaptive reference policies;
- implement hash-chained events, snapshots, resets, and deterministic replay;
- add unit and property tests for every state transition.

Later evidence, external-adapter, defense, spending, and publication work remains conditional on the
private program gates. A scaffold or reference-only effect is not evidence of an external
vulnerability.
