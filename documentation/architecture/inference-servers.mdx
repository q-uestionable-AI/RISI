title: "Standalone Inference Servers"
description: "How local, lab, or authorized cloud-hosted inference endpoints fit without becoming the primary harness"
---

## Two integration roles

A standalone inference server can provide decisions through the `DecisionProvider` boundary. In
that role it receives exact target-visible context and returns only a structured synthetic
proposal.

A server exposing persistent memory or retrieval can implement `MemoryAdapter`. In that role the
server itself can become the authorized RISI or CRAF target, while evaluator snapshots and oracles
remain outside its target-visible interface.

Deployment location does not change these roles. A cloud-hosted model can remain a
`DecisionProvider`, while a cloud-hosted service that owns persistent retrieval state must satisfy
the full `MemoryAdapter` snapshot, reset, trace, and evaluator-separation contract.

## Reserved safety profiles

`risi capabilities` reports two inference profiles as reserved and not implemented:

- `authorized-local-inference` for an exact allowlisted loopback or operator-controlled lab
  endpoint;
- `authorized-remote-inference` for an exact allowlisted public HTTPS endpoint and model, such as a
  separately authorized OpenAI-compatible deployment on Runpod.

These declarations preserve future compatibility without granting network or credential access.
Neither profile will be accepted until its controls are implemented, tested, and separately
approved. Those controls include:

- an immutable target specification covering endpoint origin and base path, model, network class,
  non-secret credential alias, and generation parameters;
- strict request, response, token, time, retry, and spend ceilings;
- DNS and resolved-IP validation plus redirect and unexpected-origin rejection;
- an explicit credential resolver that never records secret values in configuration or evidence;
- complete redacted request and response evidence, including failures and timeouts;
- operator approval bound to the exact target fingerprint and resource envelope.

## Hosted inference endpoints

The preferred future integration is a provider-neutral OpenAI-compatible decision adapter, not a
Runpod-specific control plane. This keeps the same contract usable with Runpod Serverless, an
authorized Runpod Pod, another hosted provider, or an operator-controlled local server. Provider
deployment, lifecycle management, billing administration, and arbitrary endpoint discovery remain
outside the harness.

Remote output is captured evidence, not deterministic execution. Model-free replay must verify and
rescore the retained request and response without contacting the provider; repeating a remote call
is a new run, not replay.

Application checks cannot guarantee process-level egress isolation. Local or lab inference testing
must also use suitable host, container, firewall, or VLAN controls. Remote testing additionally
requires egress restriction to the approved endpoint and connection-time enforcement of the
approved DNS, IP, TLS, redirect, and origin policy.

## Optional agent integrations

An MCP server, skill, or plugin may later expose the standalone harness to an agent ecosystem. Such
an integration must call the same application service and cannot register providers, weaken
budgets, grant capabilities, or bypass evidence generation.
